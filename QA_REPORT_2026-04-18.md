## 0. Testing Scope Caveats（先讀這個）

以下項目無法以 WebFetch + code review 方式驗證，在報告中以 `BLOCKED` 標示，**不代表沒問題也不代表沒 FAIL**，需要人類手動在真瀏覽器/裝置上做：

| 項目 | 原因 |
|---|---|
| Checkout 實際送出 | 需購物車有商品 + 登入 session，WebFetch 無 state |
| 金流實際 redirect | 需觸發 PayPal/ECPay/藍新/LINE Pay 沙箱 |
| 開發票實際 API call | 需成功訂單才觸發 ECPay invoice endpoint |
| Admin CRUD 實寫入 | 會動到 prod SQLite，需使用者明確授權才做破壞性測試 |
| iPad Safari 實機 | 無裝置；只做 code review + `/diag` 頁靜態檢查 |
| Email 實際收信 | Hetzner prod 的 SMTP 未知是否接 MailHog / SES / 其他 |
| 遊戲實際遊玩 | 多數需登入 session + 次數配額 |
| Responsive 視覺回歸 | WebFetch 無法 render；只 review Tailwind 斷點是否存在 |
